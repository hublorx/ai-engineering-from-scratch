# Poradnik forkowania

Ten kurs jest udostępniony na licencji MIT. Możesz go swobodnie forknąć i dostosować do swoich potrzeb. Oto jak zrobić to dobrze.

## Dla zespołów

Chcesz wykorzystać to jako wewnętrzne szkolenie? Skopiuj i dostosuj:

1. Skopiuj repozytorium (fork)
2. Usuń fazy, których Twój zespół nie potrzebuje
3. Dodaj przykłady i dane specyficzne dla firmy
4. Dodaj integracje z wewnętrznymi narzędziami do outputs
5. Zachowaj atrybucję — pomaga to rozwijać społeczność

## Dla szkół i uniwersytetów

Chcesz wykorzystać to jako materiał dydaktyczny?

1. Skopiuj repozytorium (fork)
2. Przypisz fazy do harmonogramu semestralnego
3. Dodaj rubryki oceniania do ćwiczeń
4. Dodaj własne zadania i egzaminy
5. Rozważ wniesienie ulepszeń do głównego nurtu projektu (upstream)

## Dla bootcampów

Prowadzisz płatny bootcamp? To jest dozwolone na licencji MIT.

1. Skopiuj i dostosuj strukturę do harmonogramu Twojej kohorty
2. Dodaj treści wideo, sesje na żywo, mentoring
3. Kod i dokumentacja są Twoje do rozbudowy
4. Rozważ sponsorowanie projektu lub wniesienie wkładu w rozwój

## Dla innych języków programowania

Chcesz nauczać tego programu nauczania w innym języku programowania?

1. Skopiuj repozytorium (fork)
2. Zaimplementuj ponownie przykłady kodu w swoim języku
3. Zachowaj strukturę lekcji i dokumentację
4. Wyślij PR, aby link do Twojego forka znalazł się w głównym README

## Utrzymywanie forka na bieżąco

```bash
git remote add upstream https://github.com/rohitg00/ai-engineering-from-scratch.git

git fetch upstream
git merge upstream/main
```
